# Design Subsystem

← [FRAMEWORK.mdc](../../.cursor/rules/FRAMEWORK.mdc)

Governs the visual language and UI/UX conventions for all projects with a user interface. Read `visual-system.md` and `tokens.md` before writing any UI code.

## Contents

| File | Purpose |
|------|---------|
| [visual-system.md](visual-system.md) | The master visual guide — layout, motion, elevation, interaction principles |
| [tokens.md](tokens.md) | Design tokens: color palette, typography scale, spacing system |
| [component-patterns.md](component-patterns.md) | Reusable UI patterns and how to apply them |

## When to Load This Subsystem

- Starting any UI feature
- Creating new components
- Reviewing design consistency
- Setting up a new project's design tokens

## Project-Level Overrides

Copy `tokens.md` into the project and fill in the brand-specific values. The visual system principles apply universally; the tokens are project-specific.
